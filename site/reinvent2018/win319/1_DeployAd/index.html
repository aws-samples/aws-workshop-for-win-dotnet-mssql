    Step 1: Deploying AD-DS on EC2 using Active Directory (AD) Quick Start

    1. Log in to your AWS account at https://aws.amazon.com/
    2. Select My Account, AWS Management Console.
    3. Enter account provided, select "Next", and then enter Username and password which will be provided separately.
    4. Create a keypair for use during this lab, Instructions are located here: https://docs.aws.amazon.com/AWSEC2/latest/WindowsGuide/ec2-key-pairs.html#having-ec2-create-your-key-pair
    5. On a new browser tab, open the Active Directory Quick Start: https://docs.aws.amazon.com/quickstart/latest/active-directory-ds/step2.html. Click Launch Quickstart for scenario 1.
    6. Make sure you select US East (N. Virginia) as your AWS region. Click Next.

    1. Provide the below inputs during the CloudFormation Stack creation:
    2. Choose at least two availability zones, preferably us-east-1e and us-east-1f.
    3. VPC CIDR: 10.0.0.0/16
    4. Private Subnet 1 CIDR: 10.0.1.0/24
    5. Private Subnet 2 CIDR: 10.0.2.0/24
    6. Public Subnet 1 CIDR: 10.0.10.0/24
    7. Public Subnet 2 CIDR: 10.0.20.0/24
    8. Allowed Remote Desktop Gateway Access CIDR: Provide your public IP address in CIDR notation. (Example: 54.239.6.185/32). You can find your public IP address by going to http://myipaddress.com using your browser.
    9. Select the keypair you created during the pre-requisite steps.
    10. Create a new keypair if you don't have the private key for an existing keypair.
    11. Select m4.xlarge for both domain controller size.
    12. Domain Controller 1 Name: onpremdc1
    13. Domain Controller 1 IP Address: 10.0.1.10
    14. Domain Controller 2 Name: onpremdc2
    15. Domain Controller 2 IP Address: 10.0.2.10
    16. .Select t2.large for Remote Desktop Gateway Instance Type
    17. Domain DNS name: Onprem.local
    18. Domain NetBIOS name: ONPREM
    19. Supply a domain admin & restore mode password and make sure to remember it.
    20. Change domain admin name from 'StackAdmin' to 'admin'.
    21. Leave Number of RDGW Hosts to 1.
    22. Leave the AWS Quick Start Configuration values for S3 Bucket Name and S3 Key Prefix to the default.
    23. Click Next to go to the next page twice and click to create the stack after selecting the check box against "I acknowledge that AWS CloudFormation might create IAM resources with custom names".
    24. CloudFormation will take some time to create this stack.
    25. To save time, you can proceed to Step 2 and come back to check the status of the CloudFormation stack. You must see CREATE_COMPLETE in the Status when the stack has finished creating.
